March 11, 2024

  • android

warning

The phone is going to be reset two times. Ressources :

  • Root All Samsung Galaxy 2023 Android 13 : to root the phone
  • Samsung Bootloder Unlock Oem Unlock Android 13/14 : to deactivate OEM

root

prepare the rooted AP file

  • connect the phone to the PC via USB
  • copy magisk apk and AP_A145RXXU1AWD1_[ . . . ]_OS13.tar.md5 to the phone
  • install magisk apk by clicking on it through the file explorer on the phone and allow installation from non-verified sources
  • launch magisk application
  • click on install button
  • select AP file
  • close application when the All done! message appears
  • copy generated magisk_patched-[ . . . ].tar file on the phone to the PC

unlock the OEM

  • activate developer mode on phone :
    • click seven times on settings->about->software information->version number, a message should appear to say the mode has been activated
  • activate OEM unlocking option in settings->development options
  • activate USB debugging option in settings->development options
  • turn off the phone
  • connect the phone to the PC through USB
  • keep pressing at the same time buttons volume up and volume down until a warning screen appears
  • press for a few seconds on volume up to turn on unlocked OEM mode
  • press volume up to confirm
  • activate USB debugging in development options when the phone is reinitialized

install the new firmware

  • turn off the phone
  • connect the phone to the PC through USB
  • keep pressing at the same time buttons volume up and volume down until a warning screen appears
  • press volume up to continue
  • verify that OEM LOCK mode is set to OFF (U)
  • launch odin on the PC :
./odin4 -a magisk_patched-[ . . . ].tar -b BL_A145RXXU1AWD1_A145RXXU1AWD1_MQB64354567_REV00_user_low_ship_MULTI_CERT.tar.md5 -c CP_A145RXXU1AWD1_CP24089644_MQB64354567_REV00_user_low_ship_MULTI_CERT.tar.md5 -s CSC_OXM_A145ROXM1AWD1_MQB64354567_REV00_user_low_ship_MULTI_CERT.tar.md5
  • if everything works fine, the PC should display a progress number and the phone a progress bar
  • follow the phone installation guide
  • copy magisk apk to the phone
  • install magisk apk by clicking on it through the file explorer on the phone and allow installation from non-verified sources
  • launch magisk application
  • verify that the button uninstall magisk is present, it confirms that the phone is rooted

setting up phone for frida use

  • activate developer mode on phone :
    • click seven times on settings->about->software information->version number, a message should appear to say the mode has been activated
  • activate USB debugging option in settings->development options
  • on the PC verify that the command adb devices returns [id] device and not [id] unauthorized
  • on the PC :
    • detect phone architecture :
    adb shell getprop ro.product.cpu.abi

arm64-v8a
    • download the corresponding frida-server file : frida github, for our version frida-server-16.1.3-android-arm64.xz
    wget https://github.com/frida/frida/releases/download/16.1.3/frida-server-16.1.3-android-arm64.xz
xz -d frida-server-16.1.3-android-arm64.xz
mv frida-server-16.1.3-android-arm64 frida-server
    • puish frida-server file to the phone :
    adb push frida-server /data/local/tmp
    • get a shell on the phone :
    adb shell
  • allow shell service to run as superuser on the phone, to do so activate com.android.shell in magisk application in Super-User window
  • execute frida-server on the PC : 
    adb shell

su
cd /data/local/tmp
chmod 755 frida-server
./frida-server
exit
exit
frida-ps -U
